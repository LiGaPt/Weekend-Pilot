# Spec: 113.5 Playwright Internal Observability E2E Teardown Exit v0

## 1. Goal

Stabilize the internal observability Playwright smoke test so a passing run exits cleanly with `exit 0` instead of hanging in teardown on Windows.

Task `113` aligned the reviewer-facing `5174` integrity surface with current evidence, but its final verification exposed a remaining infrastructure gap: the focused Playwright test body passed, yet the command timed out while Playwright was shutting down its managed backend and Vite servers. After this task, the internal observability E2E verification path must be trustworthy as a build gate: when the test passes, the process exits promptly and cleanly, without leaving orphaned servers or requiring manual cleanup.

## 2. Project Context

This task belongs to roadmap milestone `M1. 评测与观测基础设施` in `docs/NEXT_PHASE_ROADMAP.md`.

`docs/PROJECT_BLUEPRINT.md` makes benchmarkability, observability, and small reviewable verification loops core architecture principles. This task supports those principles in these areas:

- Minimal Web UI and internal observability surface verification
- Harness engineering and repeatable validation
- Deterministic operational behavior for test and review tooling
- Small, reviewable task discipline around release-readiness checks

This task is intentionally narrow. It does not add product functionality. It removes a false-negative verification failure in the current reviewer-facing E2E path.

## 3. Requirements

- The focused internal observability Playwright command must complete with `exit 0` when the test itself passes:
  - from `frontend/`: `npx playwright test e2e/internal-observability.spec.ts --project=desktop-chromium`
- The task must address teardown behavior, not test assertions.
- Managed test servers started for this command must shut down deterministically after the test run:
  - backend server on `127.0.0.1:8000`
  - customer Vite server on `127.0.0.1:5173`
  - internal Vite server on `127.0.0.1:5174`
- The shutdown path must not rely on manual intervention.
- The shutdown path must not leave orphaned backend or Vite processes after the command exits.
- `frontend/scripts/playwright-web-server.mjs` must remain the canonical wrapper for Playwright-managed server lifecycles unless implementation evidence shows a narrower fix elsewhere is sufficient.
- Add automated regression coverage for the teardown wrapper behavior.
  - The regression test must prove that a wrapped child process receives shutdown and the wrapper exits promptly.
  - The regression test must be runnable in CI/local verification without requiring a browser.
- Keep the existing Playwright test coverage for `frontend/e2e/internal-observability.spec.ts`.
- If command or script documentation is updated, it must point to the real supported verification command and not introduce parallel, conflicting E2E entrypoints.

## 6. Observability

This task does not add product observability fields.

It may add minimal test-runner diagnostics for the teardown wrapper if they are necessary to make failure modes visible during local or CI runs. Any added diagnostics must stay scoped to the E2E infrastructure and must not leak secrets or pollute normal product output.

## 7. Failure Handling

- If a managed child process ignores normal shutdown, the wrapper must escalate to a forced tree kill within a bounded timeout.
- If the wrapper cannot confirm child termination, it must exit non-zero instead of hanging indefinitely.
- If the regression teardown test detects that the wrapper leaves a child alive, the test must fail clearly.
- If the internal observability Playwright assertions fail, the command must still exit non-zero for the actual test failure reason rather than timing out in shutdown.
- If Windows-specific process-tree behavior differs from POSIX behavior, the implementation may use platform-specific shutdown handling, but the public verification command must stay the same.

## 11. Notes for the Implementer

Start from the observed failure, not from assumptions:

- In task `113`, the test body passed and Playwright reported `ok 1`, but the command later timed out while shutting down managed web servers.
- The likely fault surface is `frontend/scripts/playwright-web-server.mjs` and its interaction with Playwright `webServer` on Windows.

Keep the fix as narrow as possible. The implementer should prefer stabilizing the existing wrapper and proving it with regression coverage over introducing a new runner path or reworking all E2E scripts.

If investigation shows the primary fault is in `frontend/playwright.config.ts` or server launch strategy rather than the wrapper script itself, the implementer may adjust those files, but they must preserve one canonical focused verification command for this surface.
